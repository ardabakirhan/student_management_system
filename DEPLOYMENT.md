# Deployment Guide — Shared Hosting (cPanel + Apache + MySQL)

## 1. Import the database schema

1. Log in to **cPanel** → **phpMyAdmin**.
2. In the left panel, click **New** (or select an existing empty database).
3. Click the **Import** tab at the top.
4. Click **Choose File** and select `database/schema.sql`.
5. Leave encoding as `utf8` and click **Go**.

> The schema creates all tables and inserts the seed data in one shot.

**Re-generate password hashes (important!)**

The seed file contains placeholder bcrypt hashes. After importing, run the
following SQL in phpMyAdmin to set correct hashes for the seed users:

```sql
UPDATE users SET password_hash = '$2y$10$92IXUNpkjO0rOQ5byMi.Ye4oKoEa3Ro9llC/.og/at2.uheWG/igi' WHERE email = 'admin@rezonans.com';
-- That hash IS the bcrypt of "password" (Laravel default test hash).
-- Replace with your own hashes generated by: php -r "echo password_hash('Admin123!', PASSWORD_BCRYPT);"
```

Or use the helper script (paste into a temporary `hash.php`, visit once, then delete):

```php
<?php
$pairs = [
    'admin@rezonans.com'  => 'Admin123!',
    'selin@rezonans.com'  => 'Teacher123!',
    'burak@rezonans.com'  => 'Teacher123!',
    'zeynep@rezonans.com' => 'Teacher123!',
    'can@rezonans.com'    => 'Teacher123!',
    'fatma@rezonans.com'  => 'Veli123!',
    'mehmet@rezonans.com' => 'Veli123!',
    'ahmet@rezonans.com'  => 'Student123!',
];
// Connect and update ...
foreach ($pairs as $email => $plain) {
    $hash = password_hash($plain, PASSWORD_BCRYPT, ['cost' => 10]);
    echo "UPDATE users SET password_hash='$hash' WHERE email='$email';<br>";
}
```

Copy the generated SQL statements into phpMyAdmin and run them.

## 2. Upload the /api/ folder

Upload the entire `api/` folder to:

```
public_html/api/
```

Your hosting file tree should look like:

```
public_html/
├── api/
│   ├── config.php
│   ├── auth/
│   │   ├── login.php
│   │   ├── logout.php
│   │   └── restore.php
│   ├── users/
│   ├── students/
│   ├── evaluations/
│   ├── messages/
│   ├── announcements/
│   ├── attendance/
│   ├── payments/
│   ├── demo/
│   ├── products/
│   └── schedule/
└── (Next.js static files below)
```

Use cPanel **File Manager** or an FTP client (FileZilla, Cyberduck).

---

## 3. Set database credentials in config.php

Open `public_html/api/config.php` and replace the four constants at the top:

```php
define('DB_HOST', 'localhost');          // Almost always 'localhost' on shared hosting
define('DB_NAME', 'cpanelusername_dbname'); // Found in cPanel → MySQL Databases
define('DB_USER', 'cpanelusername_user');   // The DB user you created
define('DB_PASS', 'your_strong_password');  // The password for that user
```

Also update the `$allowed_origins` array to include your real domain:

```php
'https://yourdomain.com',
```

## 4. Upload the Next.js static build

1. Build the static export locally:

```bash
# Set your live API URL first
echo "NEXT_PUBLIC_API_URL=https://yourdomain.com/api" > .env.local
npm run build
```

This produces an `out/` folder.

2. Upload **the contents of `out/`** (not the folder itself) to `public_html/`:

```
public_html/
├── index.html
├── _next/
├── ...
```

3. Create a `.htaccess` in `public_html/` to handle Next.js client-side routing:

```apache
Options -MultiViews
RewriteEngine On

# Don't rewrite actual files or the /api/ folder
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_URI} !^/api/

# Rewrite everything else to index.html
RewriteRule ^ index.html [L]
```

## 5. Test each endpoint with curl

Replace `https://yourdomain.com/api` with your actual URL.

### Login
```bash
curl -s -X POST https://yourdomain.com/api/auth/login.php \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@rezonans.com","password":"Admin123!"}' | jq .
```
Expected: `{"success":true,"token":"...","user":{...}}`

### Save the token for subsequent requests
```bash
TOKEN=$(curl -s -X POST https://yourdomain.com/api/auth/login.php \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@rezonans.com","password":"Admin123!"}' | jq -r .token)
```

### Restore session
```bash
curl -s https://yourdomain.com/api/auth/restore.php \
  -H "Authorization: Bearer $TOKEN" | jq .
```

## Troubleshooting

| Symptom | Fix |
|---|---|
| `{"error":"Veritabanı bağlantısı başarısız."}` | Wrong DB_HOST/DB_NAME/DB_USER/DB_PASS in config.php |
| `403 Forbidden` on `/api/` | Check file permissions: PHP files should be 644, folders 755 |
| CORS errors in browser | Add your domain to `$allowed_origins` in config.php |
| `Call to undefined function str_starts_with` | PHP < 8.0 — ask host to upgrade or replace with `strpos($h,'Bearer ')===0` |
| Blank page after uploading `out/` | Check .htaccess was uploaded and mod_rewrite is enabled |
